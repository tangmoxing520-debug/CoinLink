## 一、分批止盈策略（P0）

### ✅ 实施完成

**功能描述**：
- 当持仓盈利达到10%时，止盈1/3仓位，设置保本止损
- 当持仓盈利达到20%时，再止盈1/3仓位
- 当持仓盈利达到30%时，止盈剩余全部仓位

**实施位置**：
- `trader_v2.py`:
  - `TradePosition` 数据结构添加 `initial_quantity`, `initial_margin`, `tp_level` 字段
  - `check_position()` 方法添加分批止盈检查
  - `_check_partial_take_profit()` 方法：检查分批止盈条件
  - `_partial_close_position()` 方法：执行部分平仓

**配置参数**：
```env
PARTIAL_TP_ENABLED=true
PARTIAL_TP_LEVELS=10,20,30
PARTIAL_TP_RATIOS=0.33,0.33,0.34
```

**预期效果**：
- 提高平均收益率 15-25%
- 减少"过早止盈"的遗憾
- 让利润奔跑，同时锁定部分收益

## 二、最大回撤硬止损（P0）

### ✅ 实施完成

**功能描述**：
- 当回撤达到20%时：暂停新开仓，收紧所有持仓止损到3%
- 当回撤达到30%时：完全停止交易，平仓所有持仓

**实施位置**：
- `trader_v2.py`:
  - `__init__()` 方法：初始化回撤跟踪变量
  - `_check_risk_limits()` 方法：检查最大回撤限制
  - `_close_all_positions()` 方法：紧急平仓所有持仓
  - `_get_current_equity()` 方法：获取当前权益
  - `_get_current_drawdown_pct()` 方法：计算当前回撤百分比
  - `is_trading_allowed()` 方法：检查是否允许交易
  - `monitor_positions()` 方法：在监控持仓时检查风险限制
  - `open_position()` 方法：在开仓前检查风险限制

**配置参数**：
```env
MAX_DRAWDOWN_THRESHOLD=20.0
MAX_DRAWDOWN_ACTION=pause
MAX_DRAWDOWN_SEVERE_THRESHOLD=30.0
```

**预期效果**：
- 最大回撤降低 30-40%
- 防止灾难性亏损
- 保护资金安全

## 四、市场异常检测（P0）

### ✅ 实施完成

**功能描述**：
- **闪崩保护**：如果价格在5分钟内下跌>20%，触发闪崩保护

- **流动性检测**：如果成交量突然下降>80%，触发流动性警告

  - 考虑平仓现有持仓

**实施位置**：

FLASH_CRASH_WINDOW_MINUTES=5

## 九、注意事项

1. **分批止盈**：
   - 需要确保实盘API支持部分平仓
   - 部分平仓后，剩余仓位的止损会自动调整

2. **最大回撤硬止损**：
   - 回撤计算基于最高权益点
   - 严重回撤时会自动平仓所有持仓，请谨慎使用

3. **单日亏损限制**：
   - 每日0点自动重置
   - 如果暂停交易，需要等待新的一天或手动重置

4. **市场异常检测**：
   - 闪崩检测基于价格历史，需要至少2个数据点
   - 流动性检测基于5分钟K线数据
   - 市场异常暂停后，需要手动解除（或等待30分钟自动解除）
