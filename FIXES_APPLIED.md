# 实时监控系统修复记录

## 修复内容

### ✅ P0 - 关键修复

#### 1. FOLLOW_THRESHOLD 从 30 提高到 100（与回测对齐）

**文件**：`config.env:87`

**问题**：
- 实盘逻辑只允许跟涨百分比 < 30% 的币种
- 回测逻辑允许跟涨百分比 < 100% 的币种（只要涨幅小于龙头币即可）
- 导致实盘比回测严格得多，大量有效信号被过滤

**修复**：
```diff
- FOLLOW_THRESHOLD=30
+ FOLLOW_THRESHOLD=100
```

**影响**：实盘逻辑现在与回测对齐，允许更多有效信号通过。

---

#### 2. 增强K线数据获取的重试机制

**文件**：`gate_data_fetcher.py:182-230`

**问题**：
- 数据获取失败时直接返回fallback，可能导致使用错误数据
- 数据不足时没有重试

**修复**：
- 增加额外的重试逻辑（最多2次尝试）
- 检查数据质量（至少需要10根K线）
- 失败时返回空DataFrame而不是fallback，避免使用错误数据

**影响**：提高数据获取的可靠性，减少因数据问题导致的信号丢失。

---

### ✅ P1 - 重要优化

#### 3. 放宽趋势/成交量确认条件

**文件**：`live_trader_v3.py:475-493`

**问题**：
- 趋势确认要求 `ma5 >= ma10` 且 `last_close >= ma20 * 0.98`
- 成交量确认要求 `vol_ratio >= 0.8`
- 在快速上涨初期可能过滤掉有效信号

**修复**：
- 放宽趋势确认：允许 `ma5 < ma10` 但 `last_close > ma20 * 0.95`（从0.98放宽到0.95）
- 降低成交量要求：`vol_ratio >= 0.6`（从0.8降低到0.6）

**影响**：在快速上涨初期更容易通过确认，捕获更多有效信号。

---

#### 4. 增加监控多个时间窗口

**文件**：`config.py:265`

**问题**：
- 只监控15m窗口，可能错过其他窗口的信号

**修复**：
```diff
- "time_windows": [SIGNAL_TRIGGER_INTERVAL],
+ "time_windows": ['5m', SIGNAL_TRIGGER_INTERVAL, '1h'],
```

**说明**：
- 监控多个时间窗口（5m, 15m, 1h）
- 但只有 `signal_trigger_interval`（15m）的警报才会触发交易信号
- 其他窗口的警报仅用于监控和通知

**影响**：更全面的市场监控，不会错过不同时间窗口的信号。

---

#### 5. 降低异常数据过滤阈值

**文件**：`config.py:43`

**问题**：
- `MAX_ALERT_ABS_CHANGE_5M=200` 可能允许异常数据通过
- 对于15m窗口，最大允许变化 = 200% * (15/5) = 600%

**修复**：
```diff
- MAX_ALERT_ABS_CHANGE_5M = float(os.getenv('MAX_ALERT_ABS_CHANGE_5M', '200'))
+ MAX_ALERT_ABS_CHANGE_5M = float(os.getenv('MAX_ALERT_ABS_CHANGE_5M', '100'))
```

**影响**：更严格的数据质量检查，减少异常数据导致的误报。
