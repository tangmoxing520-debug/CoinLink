# 虚拟交易模式使用指南

## 概述

虚拟交易模式是系统的默认模式，**完全不需要API密钥**，适合用于：
- 策略测试
- 参数调优
- 学习系统功能
- 验证交易逻辑

## 配置

### 基本配置

```env
# config.env
TRADE_MODE=virtual  # 虚拟模式（默认）
TRADE_ENABLED=true  # 启用交易功能
```

### 不需要配置的内容

❌ **不需要配置API密钥**
- `BINANCE_API_KEY` 可以留空或使用默认值
- `BINANCE_SECRET_KEY` 可以留空或使用默认值

✅ **系统会自动使用：**
- 币安公开API获取价格数据（无需认证）
- 虚拟资金进行交易（不影响真实账户）

## 工作原理

### 1. 数据获取

虚拟模式下，系统使用**公开API**获取数据：

- **价格数据**：`https://fapi.binance.com/fapi/v1/ticker/price`（公开接口）
- **K线数据**：`https://fapi.binance.com/fapi/v1/klines`（公开接口）

这些接口**不需要API密钥**，任何人都可以访问。

### 2. 交易执行

虚拟模式下：
- 所有交易都是**模拟的**
- 使用虚拟余额（默认20000 USDT）
- 盈亏计算基于真实价格，但**不会影响真实账户**
- 交易记录保存在本地Excel文件

### 3. 功能完整性

虚拟模式支持所有功能：
- ✅ 信号评分系统
- ✅ 止损止盈机制
- ✅ 移动止损
- ✅ 时间止损
- ✅ 黑名单机制
- ✅ 板块轮动
- ✅ 交易记录
- ✅ 钉钉通知

## 使用步骤

### 1. 启动虚拟交易

```bash
# 确保config.env中配置为虚拟模式
TRADE_MODE=virtual

# 启动系统
python live_trader_v3.py
```

### 2. 观察交易

系统会：
- 监控价格变化
- 生成交易信号
- 执行虚拟交易
- 记录交易结果

### 3. 查看结果

- **日志文件**：`crypto_monitor.log`
- **交易记录**：Excel文件（自动保存）
- **控制台输出**：实时显示交易状态

## 注意事项

### 1. 虚拟余额

- 默认虚拟余额：20000 USDT
- 可以在代码中修改初始余额
- 虚拟余额独立于真实账户

### 2. 价格数据

- 使用真实市场价格
- 数据来源：币安公开API
- 可能有轻微延迟（通常<1秒）

### 3. 交易执行

- 虚拟交易是**即时执行**的（基于当前价格）
- 不考虑滑点和手续费
- 不考虑订单簿深度

### 4. 网络要求

- 需要能够访问币安API
- 如果网络不通，系统会尝试重试
- 建议稳定的网络连接

## 常见问题

### Q1: 虚拟模式下需要配置API密钥吗？

**不需要！** 虚拟模式完全不需要API密钥。

### Q2: 虚拟交易会影响真实账户吗？

**不会！** 虚拟交易完全独立，不会影响真实账户。

### Q3: 虚拟模式下的价格是真实的吗？

**是的！** 虚拟模式使用真实的市场价格数据。

### Q4: 如何修改虚拟余额？

在代码中修改：
```python
trader = LiveTraderV3(initial_balance=50000)  # 修改初始余额
```

### Q5: 虚拟模式下的交易记录在哪里？

- Excel文件：自动保存在工作目录
- 日志文件：`crypto_monitor.log`

## 切换到实盘模式

当虚拟模式测试满意后，可以切换到实盘模式：

1. **充分测试**：在虚拟模式下至少测试1-2周
2. **配置API密钥**：按照 `REAL_TRADING_GUIDE.md` 配置
3. **修改配置**：
   ```env
   TRADE_MODE=real
   ```
4. **小资金测试**：先用小资金测试实盘功能
