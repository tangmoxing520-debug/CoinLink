# 实盘交易安全使用指南

## ⚠️ 重要警告

**实盘交易涉及真实资金，存在高风险！请务必仔细阅读本指南并充分测试后再启用实盘交易！**

## 1. 准备工作

### 1.1 虚拟模式 vs 实盘模式

**虚拟模式（默认，推荐用于测试）：**
- ✅ **不需要API密钥**
- ✅ 使用虚拟资金进行交易
- ✅ 使用公开API获取价格数据（无需认证）
- ✅ 完全安全，不会影响真实账户

**实盘模式（需要谨慎）：**
- ⚠️ **必须配置API密钥**
- ⚠️ 使用真实资金进行交易
- ⚠️ 需要API密钥访问交易接口

### 1.2 API密钥配置（仅实盘模式需要）

**虚拟模式下可以跳过此步骤！**

1. **创建币安API密钥**
   - 登录币安账户
   - 进入 API管理页面
   - 创建新的API密钥
   - **重要：只授予"启用交易"权限，不要授予"提现"权限！**

2. **配置config.env（仅实盘模式）**
   ```env
   # 交易模式配置
   TRADE_MODE=real  # 改为real启用实盘交易（virtual=虚拟模式，不需要密钥）
   TRADE_ENABLED=true
   
   # 币安 API 密钥（仅实盘模式需要）
   BINANCE_API_KEY=your_actual_api_key_here
   BINANCE_SECRET_KEY=your_actual_secret_key_here
   ```

### 1.2 安全建议

- ✅ **使用独立的API密钥**：不要使用主账户的API密钥
- ✅ **限制IP访问**：在币安API设置中限制IP白名单
- ✅ **设置交易限额**：在币安账户中设置每日交易限额
- ✅ **启用2FA**：确保账户启用双因素认证
- ✅ **定期检查**：定期检查API密钥的使用情况

## 3. 风险控制

### 3.1 交易限额

系统内置了以下保护机制：

- **单笔最大交易金额**：5000 USDT（可在代码中调整）
- **单日最大交易金额**：50000 USDT（可在代码中调整）
- **最大持仓数量**：8个（V6参数）

### 3.2 止损保护

系统使用多重止损机制：

- **固定止损**：4%价格变化（杠杆后-60%）
- **移动止损**：1.5%回撤（激活阈值2.5%）
- **保本止损**：2%盈利后激活
- **时间止损**：短期1.5小时，长期8小时

### 3.3 黑名单机制

- **静态黑名单**：预设表现差的交易对
- **动态黑名单**：连续亏损5次自动加入
- **分类黑名单**：禁止交易的板块

## 5. 常见问题

### Q1: 虚拟模式下需要配置API密钥吗？

**不需要！** 虚拟模式完全不需要API密钥。

虚拟模式下：
- 使用虚拟资金（不会影响真实账户）
- 使用公开API获取价格数据（无需认证）
- 所有交易都是模拟的

只需要配置：
```env
TRADE_MODE=virtual
TRADE_ENABLED=true
```

### Q2: 如何从实盘模式切换回虚拟模式？

```env
# 修改config.env
TRADE_MODE=virtual
# API密钥可以保留或删除（虚拟模式下不会使用）
```

重启程序即可。

### Q2: 实盘交易时如何暂停交易？

```env
# 修改config.env
TRADE_ENABLED=false
```

系统会继续监控，但不会执行交易。

### Q3: API密钥验证失败怎么办？

1. 检查API密钥是否正确配置
2. 检查API密钥权限（需要"启用交易"权限）
3. 检查IP白名单设置
4. 检查网络连接

### Q4: 订单执行失败怎么办？

1. 检查余额是否充足
2. 检查交易对是否支持
3. 检查杠杆设置是否合理
4. 查看日志了解具体错误
